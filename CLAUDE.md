## Site Overview

Personal portfolio and blog hosted on GitHub Pages. No build tools, bundlers, or server required — open `index.html` directly in a browser or use VS Code Live Server.

## Development

**Preview locally:** Open `index.html` in a browser (no build step needed).

**Adding a blog post:**
1. Create `blogs/posts/<slug>.md` with YAML frontmatter:
   - Required: `title`, `date` (YYYY-MM-DD), `excerpt`, `thumbnail`
   - Optional: `short_title` (card display), `permalink` (custom URL slug)
2. Run: `python3 convert-blog.py blogs/posts/<slug>.md --update-index`
   - Requires: `pip install pyyaml`
   - Creates `blogs/<slug>.html` and updates `blogs/blogs.yaml`
3. Blog images go in `blogs/images/`

**Other converter options:**
- `python3 convert-blog.py --convert-all` — regenerate all blog HTML from markdown sources

## Architecture

**Single-file philosophy:** All CSS is in `css/style.css` and all JavaScript is in `js/main.js`. Both files use clearly marked section headers (`/* ====== SECTION ====== */` and `/** === SECTION === */`) so you can navigate by searching.

**Content vs. code separation:**
- Site content (bio, publications, timeline, media, design tokens) lives in `content/` as YAML/markdown — no code changes needed for content updates
- Blogs live in `blogs/` (not `content/blogs/`) to produce clean shareable URLs

**Key files:**
| File | Purpose |
|------|---------|
| `content/site/config.yaml` | Single source of truth for site name, nav, social links, and entire design system (colours, fonts, spacing) |
| `content/site/bio.md` | Profile bio (markdown) |
| `content/publications/publications.yaml` | Publications list |
| `content/timeline/timeline.yaml` | Work/education timeline |
| `content/media/media.yaml` | Media highlights (YouTube, etc.) |
| `blogs/blogs.yaml` | Blog index — auto-generated by `convert-blog.py --update-index` |
| `blogs/_blog_template.html` | Template used by the converter to produce blog HTML files |
| `js/main.js` | All JS: config loading, YAML fetching, markdown parsing, routing, theme toggle, TOC generation |
| `css/style.css` | All CSS: variables (light/dark themes), layout, components, responsive rules |

**JavaScript structure (`js/main.js`):** Wrapped in an IIFE. Sections in order: config loader → utility functions → timeline → media → main page loaders (publications, blogs, profile) → blog page functions → page detection/init.

**CSS structure (`css/style.css`):** CSS variables at the top define the entire design system (colours, spacing, sizes) for both light and dark themes. Edit these variables for quick global changes.

**Blog HTML generation:** `convert-blog.py` embeds blog content directly into each HTML file as `window.BLOG_POST_DATA` and injects Open Graph meta tags from the YAML frontmatter. This makes each blog post self-contained with no runtime fetches for content.
